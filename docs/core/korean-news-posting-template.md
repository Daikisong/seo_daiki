# Korean News Posting Template

이 문서는 한국어 `news` 글을 만들 때 따라야 할 공개 글 기준을 정리한다.

목표는 내부 분석 화면을 예쁘게 보여주는 것이 아니다. 독자가 검색해서 들어왔을 때 바로 읽을 수 있는 기사형 글을 만드는 것이다.

## When To Use

`news` 분기는 상품 링크가 어색한 정보형 트렌드에 쓴다.

예:

```text
교육공무원
KBO 올스타 팬투표
정부 지원금
세금 신고
공공 서비스 변경
학교/입시/제도 변경
```

구매나 비교 의도가 분명하면 `review` 분기로 보낸다.

예:

```text
게이밍 모니터 추천
아이폰 할인 비교
보조배터리 구매 가이드
TV 할인 비교
```

## Public Article Flow

한국어 뉴스 글의 공개 화면은 단순해야 한다.

```text
1. 제목과 요약
2. 이 글의 요점
3. 본문 목차
4. 본문 섹션
5. 하단 출처와 정정 안내
6. 이전/다음 글
```

예를 들어 `교육공무원` 글이면 상단에 내부 체크 화면을 놓지 않는다. 대신 본문에서 “교육공무원과 교육공무직은 먼저 구분해야 한다”처럼 독자가 이해할 문장으로 풀어 쓴다.

## Data Contract

공개 뉴스 글에서 필요한 기본 데이터는 다음이다.

```text
title
summary
sections
heroImage
articleMeta
keyTakeaways
sourceLinks
internalLinks
contentBranch = news
monetizationRoute = informational_explainer
indexStatus = noindex 또는 승격 후 index
```

뉴스 글은 공개 화면에 별도 관리용 패널 데이터를 전달하지 않는다. 체크해야 할 항목이나 용어 설명이 필요하면 `sections` 본문 안에 자연스럽게 넣는다.

예:

```text
나쁜 방식:
  상단에 관리용 체크 박스와 내부 판단 목록을 길게 보여준다.

좋은 방식:
  본문 한 섹션에서 “지원 전에는 공고명, 채용 구분, 접수 기간을 순서대로 확인해야 한다”라고 설명한다.
```

## Writing Rules

한국어 뉴스 글은 다음 원칙을 지킨다.

```text
1. 첫 섹션에서 독자가 지금 무엇을 확인해야 하는지 말한다.
2. 공식 자료가 중요한 주제는 하단 출처에 확인일을 남긴다.
3. 검색어 하나만 보고 제도 변경이나 일정 변경을 단정하지 않는다.
4. 법률, 보수, 지원 자격, 정책 효과는 공식 문서 확인을 전제로 쓴다.
5. 상품 링크와 제휴 CTA를 넣지 않는다.
6. 내부 작업 상태나 파이프라인 판단은 본문에 쓰지 않는다.
```

금지하는 공개 표현 예:

```text
테스트 포스트
상품 후보 pending
수익화 보류
SERP 분석
트렌드 신호
상위 글을 참고함
```

## Korean Example Shape

`교육공무원` 같은 글은 다음처럼 만든다.

```text
제목:
  교육공무원 2026: 채용·복무·보수 확인 전에 볼 공식 경로

요점:
  법령, 채용 공고, 보수표를 한 검색어로 섞어 보면 안 된다.

본문 섹션:
  빠른 결론
  왜 먼저 구분해야 하나
  교육공무원과 교육공무직을 먼저 구분해야 한다
  법령은 공식 법령 페이지에서 시행일을 본다
  채용 공고는 시도교육청별로 다르다
  보수와 수당은 숫자 하나로 판단하지 않는다
  지원 전에 꼭 구분할 것
  지원 전 확인 순서

하단 출처:
  법령 자료
  교육청 공고
  보수 기준 자료
```

## Acceptance

한국어 뉴스 글 하나를 만들 때 최소 기준:

```text
sections >= 6
keyTakeaways >= 3
sourceLinks >= 3
sourceLinks checkedAt 값 존재
heroImage alt/caption 존재
본문에 내부 작업 상태 없음
상단에 관리용 패널 없음
pnpm seo:article-quality 통과
```
