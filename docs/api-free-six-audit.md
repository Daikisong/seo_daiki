# API-Free Six Flow Audit

## Scope

This audit covers the API-free flow in `docs/goal.md`:

```text
trend detection -> keyword generation -> manual SERP observation -> competitor structure analysis -> dry-run strategy -> noindex test post
```

It intentionally excludes Search Console, product candidate analysis, monetization, affiliate APIs, distribution, and outreach.

## Current Commands For Steps 1-6

| Step | Command | Current source |
| --- | --- | --- |
| Detect market trends | `trend:init-markets`, `trend:import-signals`, `trend:normalize`, `trend:cluster`, `trend:score` | `workers/python/intelligence/market_trend_engine.py` |
| Generate trend keywords | `trend:generate-keywords` | `workers/python/intelligence/market_trend_keywords.py` |
| Observe SERP results | `serp:import-results`, `serp:fetch-pages` | `workers/python/serp/serp_importer.py` |
| Analyze top page structures | `serp:analyze-pages`, `serp:summarize-opportunity` | `workers/python/serp/serp_page_analysis.py`, `workers/python/serp/serp_opportunities.py` |
| Decide article strategy | `strategy:create`, `strategy:generate-brief` | `workers/python/writers/market_content_strategy_records.py` |
| Generate and publish test posts | `post:generate-test`, `post:publish-test --mode noindex` | `workers/python/writers/market_test_articles.py` |

## Commands That Require APIs

The API-free six flow does not require any external API credentials.

Excluded later-phase commands include:

- Search Console performance import
- product candidate discovery/analysis
- monetization review
- live affiliate APIs
- posting/distribution/outreach
- offer matching

## Commands Using Seed CSV / Local JSON

- `trend:import-signals` reads `data/seeds/trend-signals.csv`.
- `serp:import-results` reads `data/seeds/serp-results.csv`.
- `serp:analyze-pages` reads `data/seeds/competitor-page-summaries.csv`.
- Strategy and post steps read local JSON generated by earlier steps.

## Breaks Found

Before this patch, the pieces existed but the goal-specific command did not:

- no `pipeline:api-free-six` package script
- no `verify:api-free-six` package script
- no dedicated `pipeline_api_free_six_run.json`
- `pipeline:trend-to-post` built calendars and did not publish noindex test posts as part of the exact API-free six flow
- no success-criteria verifier for generated output counts, route paths, noindex metadata, and safety invariants
- test post page did not display `monetizationDeferred`

## Web Routes

Generated test posts should render at:

```text
/[market]/[language]/posts/[slug]/
```

Examples:

```text
/us/en/posts/magnesium-sleep/
/es/es/posts/usb-c-charger/
/br/pt-br/posts/power-bank-real-capacity/
/jp/ja/posts/compact-desk-gadget/
/kr/ko/posts/gut-health/
```

The post route reads `data/exports/test_articles.json` through `readMarketPosts()`.
