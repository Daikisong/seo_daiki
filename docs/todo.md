### 현재 판단

우리 웹사이트는 SEO 구조의 방향은 좋다.

```text
시장별 silo
review/news 분기
상단 신뢰 정보
바로 답 / 핵심 요약
목차
체크리스트
비교표
출처 카드
내부 연구 문구 비공개
```

하지만 아직 여러 나라 트렌드 상위권 사이트급은 아니다.

쉽게 말하면:

```text
상위 사이트:
  공식 데이터, 실시간성, 브랜드 권위, 오래 쌓인 내부 링크, 주제별 콘텐츠 깊이가 있음.

우리 사이트:
  화면 구조는 좋아졌지만 실제 게시물은 7개, 커버 마켓은 5개뿐이고,
  18개 마켓별 topical authority와 실시간 데이터/공식 연동은 아직 부족함.
```

예를 들어 `KBO 올스타 팬투표`는 우리가 보기 좋게 정리할 수는 있지만, KBO 공식 사이트나 네이버 스포츠처럼 공식성/속보성/브랜드 권위가 있지는 않다. 그래서 직접 1차 출처를 이기려 하기보다 `후보 확인 방법`, `일정`, `투표 전 체크할 것`, `공식 링크`처럼 정리형 보조 검색 의도를 노리는 게 현실적이다.

### 해야 할 작업

1. 18개 마켓에 실제 현재 트렌드 글을 최소 1개씩 만든다.
2. 각 트렌드마다 `review` 또는 `news` 분기를 먼저 판단한다.
3. 뉴스형 글에 다음 모듈을 추가한다.

```text
상태 카드
타임라인
공식 링크 박스
무엇이 바뀌었는지
누가 영향을 받는지
다음에 확인할 것
출처 최신 확인 날짜
```

4. 리뷰/비교형 글에는 다음 모듈을 강화한다.

```text
비교표
구매 전 체크리스트
점수/검토 방식
제품 이미지
가격/출처 확인 날짜
대안 제품
상품 후보 분석
```

5. 날씨, 주가, TV 편성, 스포츠 일정, 항공 상태, 금 가격 같은 주제는 live data 없이 정면 경쟁하지 않는다.

예:

```text
나쁜 방향:
  "TSX today"를 실시간 데이터 없이 작성

좋은 방향:
  "TSX가 오늘 검색되는 이유와 공식 지표를 확인하는 방법"
```

6. 각 시장별로 주제 클러스터를 만든다.

최소 구조:

```text
메인 트렌드 글 1개
보조 설명 글 2개
공식 출처 확인용 체크리스트 1개
제품형이면 비교/FAQ 글 1개
```

7. index 전환 기준을 만든다.

```text
test/noindex
  -> 출처/형식/품질 검증
  -> market별 글 깊이 확보
  -> 내부 링크 연결
  -> index 후보로 승격
```

### 우선순위

P0:

```text
18개 마켓 각각 현재 트렌드 글 1개 생성
뉴스형 상세 페이지에 상태 카드/공식 링크 박스 추가
review/news 분기 판단 로직 유지
```

P1:

```text
마켓별 3-5개 supporting article cluster 만들기
리뷰형 글의 비교표/상품 후보 분석 강화
시장별 내부 링크 자동 추천
```

P2:

```text
날씨/주가/스포츠/TV 편성 같은 live-data 주제는 provider 계약 또는 수동 공식 링크 기반으로만 확장
noindex -> index promotion workflow 정리
```
